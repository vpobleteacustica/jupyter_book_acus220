Crear un proyecto en un repositorio nuevo.¶

  • Entrar a su cuenta en GitHub y crear un repositorio nuevo.

  • Darle un nombre al repositorio Git.

    • Por ejemplo, le llamaremos uso_git.

  • Redactar una descripción inicial.

    • Por ejemplo: Uso básico de Git y primeras aplicaciones.

  • Elegir entre Privado o Público.

    • Sugerencia seleccionar Público.

  • Agregar un archivo README.

    • Este es un instructivo global del repositorio que se está creando.

  • Agregar un .gitignore

    • Este es importante porque es aquí donde uno declara qué archivos o carpetas, uno no quiere que se suban al repositorio Git, que se mantengan de manera local solamente. Como por ejemplo, las carpetas y archivos del entorno virtual ya que uno no quiere que esas instalaciones, que uno tiene en local, se suban al repositorio Git.

    • Observar las opciones que ofrece. Dependiendo del lenguaje y del proyecto, uno puede elegir un .gitignore en particular, como por ejemplo, Python, que tiene ya una lista de archivos posible que uno no quiere subir.

  • Omitir por esta vez Choose a license.

  • Crear el repositorio.

Observar lo que se muestra en el repositorio.¶

  • El Readme que se ve al principio.

  • El .gitignore para Python que se seleccionó, que muestra archivos y carpetas que se generaron en Python. Por ejemplo, se observan los # Jupyter Notebook, # Environments, entre otros.

  • Este archivo .gitignore es totalmente editable.

  • Ir a main

Ramas en el repositorio.¶

  • Siempre al principio en el repositorio hay una sola rama que se llama main, que es la principal.

  • Vamos a traer este repositorio Git que hemos creado a nuestra máquina.

  • Crear en el escritorio de su máquina una nueva carpeta. Por ejemplo, le llamaremos github.

  • Ya en en su nueva carpeta, abrir una terminal.

  • Copiar la dirección del repositorio que acabamos de crear en GitHub y en la terminal abierta, ejecutemos la línea de comando:

kasparov@kasparov-MS-7B58:~/Escritorio/github$ git clone <link del repositorio GitHub>

Entrar a la carpeta.¶

  • En la terminal abierta, vamos a entrar a la carpeta ejecutando la línea de comando:

kasparov@kasparov-MS-7B58:~/Escritorio/github$ cd uso_git

  • El único archivo que se ve es el README.md. Se debería ver lo siguiente:



  • El archivo .gitignore está oculto. Para ver los archivos ocultos ejecutar la línea de comando:

kasparov@kasparov-MS-7B58:~/Escritorio/github/uso_git$ ls -la

  • Si todo está bien, deberíamos ver:



  • También, vemos la carpeta .git. Sin esta carpeta, no se podría hacer por ejemplo, lo siguiente:

kasparov@kasparov-MS-7B58:~/Escritorio/github/uso_git$ git status

  • Lo que nos permite ver que esto es un proyecto Git. Siempre para ver el estado de un repositorio usamos git status. Nos muestra en qué rama estamos y que esta rama está actualizada.
